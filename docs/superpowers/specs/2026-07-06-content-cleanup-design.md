# Этап 1: Контент-чистка сайта Kucher&Conga

Дата: 2026-07-06. Первый из четырёх этапов: **контент-чистка → визуальный редизайн → формы заказа/брони → SEO/GEO-оптимизация**.

## Цель

Убрать с сайта всю неактуальную, тестовую и моковую информацию, заменить актуальными данными от владельца, добавить изображения в разделы без картинок, встроить опросник отклика на вакансии с записью в Google Таблицу.

## 1. Hero: карусель «Новые блюда» вместо НГ-баннера

- Убрать захардкоженный `/kongo_ng.webp` (HomeClient.tsx:750).
- На его месте — карусель из 5 изображений «Новые блюда» (файлы из Downloads от 06.07, jpg → webp, в `public/`): обложка, закуски и стартеры, основные блюда I, основные блюда II, десерты.
- Поведение: автопрокрутка (~5 с), свайп на мобильных, стрелки/точки на десктопе. Клик никуда не ведёт (страница /events пустая после чистки) — карусель самодостаточна.

## 3. Вакансии: 5 актуальных позиций + опросник

### Контент
Удалить тестовые записи из Supabase `content_posts` (category=vacancies). Залить 5 актуальных вакансий (тексты от владельца, 06.07.2026):

| Позиция | Оплата | График |
|---|---|---|
| Официант | 4000 ₽/смена + чаевые | гибкий |
| Администратор ресторана | 6000 ₽/смена | 5/2 |
| Повар горячего цеха | от 5000 ₽/смена | 5/2 |
| Бармен-кассир | 4000 ₽/смена | гибкий |
| Кальянщик | 3000 ₽/смена + 10% от кальяна | 5/2 |

**Без кнопок «Позвонить HR» / «Написать в Telegram»** — единственный канал отклика: анкета на сайте.

### Опросник отклика
- На странице вакансии кнопка «Откликнуться» → форма (расширенный набор полей): ФИО, телефон, возраст, гражданство, желаемая вакансия (автоподставляется), опыт работы (где/сколько), медкнижка (есть/нет/готов оформить), когда готов выйти, ожидания по зарплате, ссылка на резюме или Telegram, готовность к графику, комментарий о себе.
- Отправка: `POST /api/vacancy-apply` → Google Apps Script Web App → Google Таблица на Диске владельца.
- Лист на каждый год создаётся автоматически («2026», «2027»...), строка = отклик, шапка с названиями колонок, форматирование при создании листа.
- Владелец создаёт таблицу и вставляет готовый Apps Script по инструкции (будет в `docs/`); URL скрипта — в env `VACANCY_SHEETS_WEBHOOK_URL`.
- Валидация на сервере: обязательные ФИО + телефон, лимиты длины. При недоступности скрипта — ответ пользователю «попробуйте позже» (без тихой потери данных).
